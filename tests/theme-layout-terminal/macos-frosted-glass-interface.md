# macOS 磨砂玻璃简白界面与手机触屏可用性

## 功能/改动名称

macOS 磨砂玻璃简白视觉层、展开界面层级修复和粗指针触屏可用性。

## 前置条件/准备

1. 在当前工作树运行 `pnpm run dev --host 127.0.0.1 --port 4173`。
2. 准备一个包含 Git 状态、文件改动、终端命令和多代理进度的现有线程。
3. 分别使用 `375x812`、`767x900`、`768x1024` 和 `1440x900` 视口。
4. 在前三个视口启用触屏/粗指针模拟；手机操作必须使用真实 `tap`，不得使用 `hover` 或强制点击。
5. 每个视口分别验证浅色和深色主题。

## 操作步骤

1. 打开首页，检查画布、侧栏或手机抽屉、顶栏、主内容和 Composer 的简白半透明表面、细边框、内高光与阴影。
2. 在手机端打开侧栏，确认线程时间与“…”菜单同时可见，再依次打开并关闭设置、整理线程、项目菜单和线程菜单；把“整理线程”按钮滚动到侧栏底部附近，确认菜单自动向上展开；点击菜单外部确认它们关闭。
3. 在设置中展开账户区、界面语言下拉、听写语言下拉和 Telegram 配置；滚动面板并确认每个控件可触达。
4. 在 Composer 中依次打开权限、Skills、模型/推理和模型二级菜单；确认菜单完全落在视口内并可关闭。
5. 打开“创建项目”弹窗，切换“新项目”和“从 GitHub 克隆”；打开“导入项目”文件选择器但不选择文件。
6. 在线程页打开 Git 菜单；在 `767` 和 `768` 宽度选择一个可读取提交的分支，再选择一条提交，确认文件、提交、分支三栏均在视口内。
7. 从顶栏打开终端，确认终端面板出现且 Composer 仍可用，然后仅隐藏终端，不关闭 PTY。
8. 打开自动化创建弹窗，滚动到状态下拉；确认底部操作区不遮挡状态控件，状态菜单向上展开，随后取消。
9. 打开一个 Skill 详情和一个插件详情；在 `375x812` 下确认 Skill 最多四个底部按钮以两列换行且全部可点击。
10. 打开归档聊天的永久删除确认框，只检查取消和删除按钮，随后取消，不执行删除。
11. 给 Composer 临时附加一张图片，打开图片预览后关闭并移除附件，不发送消息。
12. 在线程消息中展开文件改动，打开 Diff；手机端再打开文件列表 sheet，关闭 sheet 和 Diff。
13. 从 Git 菜单打开 Review；手机端打开文件列表 sheet，关闭 sheet 和 Review。
14. 打开手机多代理进度 sheet，确认遮罩覆盖顶栏且顶栏不可点穿，然后关闭。
15. 在 Composer 输入至少七行文本，打开全屏输入模式，再退出并清空草稿。
16. 对每个已打开的菜单、sheet 和弹窗检查：边界位于 viewport 内，中心点命中自身或后代，同时只存在一个同类展开层，关闭后容器消失。

## 预期结果

- 浅色主题呈低饱和蓝灰渐变画布、半透明简白表面、细边框和柔和阴影；深色主题使用成对的石墨玻璃语义色。
- 主题不会改变桌面网格、侧栏宽度、滚动结构或组件定位，也不会通过滤镜创建新的 fixed 定位参照。
- 菜单、弹窗、输入框、代码、Diff 和终端保持高不透明度；手机抽屉不会透出底层正文。
- 图片、Diff、Skill、Directory、Composer 全屏和多代理进度层遮住顶栏，不能点击穿透。
- 线程时间保持可见，线程菜单无需 hover 即可触达；项目、线程、整理和顶栏按钮的粗指针命中区互不覆盖；整理菜单在下方空间不足时向上展开且所有选项可点。
- 所有输入控件在手机端至少使用 16px 字号，不触发 iOS 自动缩放。
- 所有被打开的原有功能仍可执行或安全取消；不出现横向页面溢出、无法点击或展开层残留。

## 回滚/清理

1. 清空 Composer 草稿并移除临时图片附件。
2. 隐藏测试时打开的终端、关闭所有菜单、sheet 和弹窗，并恢复测试前的主题。
3. 不保存项目、自动化、重命名或删除操作；若为验收创建了临时实体，仅删除对应临时实体。
4. 若需回滚本改动，恢复本次提交涉及的主题和触屏修复文件，并重新运行本用例确认基线行为。
